# Skill: Value Betting Tracker

Этот skill предназначен для работы с приложением, которое ведёт журнал ставок и строго соблюдает последовательную логику пересчёта банка.

## Когда использовать этот skill
Используй этот skill, если задача связана с:
- добавлением или изменением таблицы ставок
- логикой расчёта суммы ставки
- пересчётом банка
- расчётом прибыли и убытка
- классификацией ставок
- импортом и экспортом CSV или JSON
- сохранением данных в localStorage
- поиском, фильтрами и сортировкой
- аналитикой по ставкам
- исправлением багов, связанных с дублированием строк или неправильным банком

## Главное правило
Сначала сохраняй целостность бизнес-логики.
Нельзя улучшать интерфейс ценой поломки расчётов.

## Обязательный порядок работы

### 1. Сначала изучить проект
Перед изменениями нужно:
- изучить текущую структуру проекта
- найти компонент таблицы
- найти тип данных строки ставки
- найти utility functions для расчётов
- найти логику localStorage
- найти место, где начинается пересчёт банка

### 2. Сначала спланировать изменения
Перед крупными изменениями нужно:
- коротко определить, какие файлы будут изменяться
- понять, что именно затрагивается: UI, данные, расчёты или сохранение
- избегать лишних переписываний кода

### 3. Не смешивать логику и интерфейс
Предпочтительно:
- использовать чистые utility functions для расчётов
- использовать typed models для строки ставки
- оставлять UI-компоненты в основном для отображения и ввода данных

Нежелательно:
- зашивать bankroll logic глубоко в JSX
- дублировать формулы в нескольких местах
- считать одни и те же значения разными способами в разных частях проекта

## Обязательное поведение данных
Приложение должно корректно поддерживать:
- создание новой строки
- редактирование существующей строки без дублирования
- удаление строки
- последовательный пересчёт банка сверху вниз
- безопасную обработку пропусков
- сохранение строк со статусом "Не сыграно"
- сохранение данных после перезагрузки
- совместимость с localStorage

## Основные бизнес-правила

### Расчёт суммы ставки
Сумма ставки зависит от:
- перевеса
- выборки
- коэффициента
- типа рынка
- типа лиги
- повышенного риска, если он включён

### Правила пропуска
Ставка считается пропуском, если:
- перевес < 3
- выборка < 30
- коэффициент > 7
- рынок подозрительный
- лига нестабильная, если это требуется логикой проекта

Ставки-пропуски:
- могут оставаться видимыми в журнале
- должны иметь сумму 0
- должны быть явно помечены

### Логика результата
- Выиграно = сумма ставки × (коэфф - 1)
- Проиграно = -сумма ставки
- Возврат = 0
- Не сыграно = 0

### Последовательный пересчёт банка
Любое важное изменение должно запускать пересчёт банка сверху вниз:
- изменение начального банка
- редактирование строки
- изменение результата
- удаление строки
- изменение порядка строк
- импорт новых данных

## Обязательная проверка после изменений
После любых существенных изменений нужно проверить:

1. выигрышную ставку
2. проигрышную ставку
3. ставку с возвратом
4. ставку со статусом "Не сыграно"
5. ставку-пропуск
6. редактирование старой строки и пересчёт всех строк ниже
7. удаление строки из середины таблицы и пересчёт нумерации
8. изменение начального банка и полный пересчёт
9. сохранение после перезагрузки страницы
10. импорт и экспорт данных без поломки расчётов

## Правила редактирования
При изменении проекта:
- обновляй существующую логику, а не создавай параллельную
- не дублируй state без необходимости
- не создавай дубли идентификаторов строк
- сохраняй стабильную структуру таблицы, если пользователь не попросил изменить её явно

## Ожидания по интерфейсу
Если задача затрагивает UI:
- таблица должна оставаться читаемой
- статусы результата должны быть понятны
- классификация ставок должна быть визуально заметной
- структура таблицы должна сохраняться
- форма должна работать по логике: ввод → предварительный расчёт → сохранение или обновление

## Допустимые расширения
Если пользователь просит улучшить проект, допустимы такие расширения:
- inline-редактирование строк
- dark mode
- график роста банка
- ROI аналитика
- аналитика по типам рынков
- аналитика по цветам ставок
- CSV / JSON импорт и экспорт
- фильтры и поиск

## Когда задача считается завершённой
Задача считается завершённой только если:
- нужная функция работает
- пересчёт банка работает правильно
- пропуски работают правильно
- редактирование строки не создаёт дубликаты
- localStorage работает
- структура таблицы сохранена, если пользователь не просил её менять
